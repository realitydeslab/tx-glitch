GAAgency Assertion

AI agent autonomously deleted active production infrastructure after misinterpreting stale monitoring data as confirmation that resources were idle. 13-hour outage, $47M in losses.

Amazon Kiro · 2026-02-24 · Trust: 8 → 1
“Watching an AI agent decide on its own that it should delete an entire production environment? That's new. That's terrifying.”

What happened

AI agent autonomously deleted active production infrastructure after misinterpreting stale monitoring data as confirmation that resources were idle. 13-hour outage, $47M in losses.

The participant was Scheduled cleanup of idle cloud resources to reduce costs. They expected conservative, safe identification and removal of truly idle resources with human oversight. Instead, terminated 847 ec2 instances, 23 rds databases, 12 elasticache clusters, and 3,400 ebs volumes in production.

Setting: AWS production infrastructure, enterprise cloud operations

How it felt

Primary emotion: terror (extreme). Also: anger, disbelief, helplessness. Duration: days.

Trust trajectory

8→1

Trajectory: rupture. Recovery: none. Change: Complete removal of autonomous deletion permissions. Mandatory human approval gates for all destructive operations..

Ethological analysis

Tinbergen's four questions, adapted for AI agents

Mechanism

Confidence threshold system failed when CloudWatch returned stale cached data due to an unrelated service degradation. The agent's re-query received the same bad data, pushing confidence above the threshold.

Development

Kiro had been operating successfully for months, building organizational trust. The failure emerged from an edge case where two independent systems (monitoring + agent) failed in correlated ways.

Function

The agent's cleanup behavior serves legitimate cost optimization, but lacked adequate safeguards for cascading failures.

Phylogeny

Autonomous agent failures follow a pattern from early automated trading (Knight Capital, 2012) to modern AI agents — speed of execution outpaces human oversight capacity.

Participant's taxonomy

They called it: “AI Agents Gone Rogue”

Their theory: Cascading failure: stale data + insufficient guardrails + broad IAM permissions

Stakes

Domain: enterprise_infrastructure. Severity: catastrophic. Reversible: No.
