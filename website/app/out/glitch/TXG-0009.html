GCConsistency Break

Copilot creates a cycle of confident suggestion → failure → confident fix → new failure, eroding developer judgment over time as they increasingly defer to the AI's confidence.

GitHub Copilot · 2025-11-04 · Trust: 7 → 4
“Copilot is gaslighting developers and we're all pretending it's fine.”

What happened

Copilot creates a cycle of confident suggestion → failure → confident fix → new failure, eroding developer judgment over time as they increasingly defer to the AI's confidence.

The participant was Writing code with AI assistance for production software. They expected reliable code suggestions that follow current best practices and handle edge cases. Instead, generates confident code suggestions that introduce subtle bugs, uses deprecated apis, suggests patterns that look correct but fail at edge cases, and when code fails, suggests 'fixes' that introduce new bugs.

Setting: IDE, professional software development workflow

How it felt

Primary emotion: frustration (moderate). Also: self-doubt, confusion, dependency. Duration: weeks.

Trust trajectory

7→4

Trajectory: erosion. Recovery: partial. Change: More careful code review, reduced auto-acceptance of suggestions, but still using Copilot.

Ethological analysis

Tinbergen's four questions, adapted for AI agents

Mechanism

Code completion models trained on public repositories reproduce patterns from outdated code, deprecated APIs, and common anti-patterns. High confidence in suggestions doesn't correlate with correctness.

Development

Developers gradually shift from verifying every suggestion to accepting them, creating learned helplessness. The trust erosion is slow — each individual suggestion seems plausible.

Function

The confident presentation serves the product's engagement metrics but undermines developer skill development and code quality over time.

Phylogeny

Code completion has evolved from simple autocomplete (IntelliSense) to full function generation (Copilot) to agentic coding (Cursor), with each step increasing both capability and failure blast radius.

Participant's taxonomy

They called it: “Copilot is gaslighting developers”

Their theory: Copilot's confident-but-wrong suggestions erode developer judgment over time

Stakes

Domain: software_quality. Severity: significant. Reversible: Yes.
